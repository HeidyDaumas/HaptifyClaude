Platform: macOS (Force Touch trackpad)

01 Overview

HaptiClaude is a Claude Code plugin that triggers MacBook Force Touch trackpad haptic feedback whenever a Claude Code session needs user attention. Never miss a permission prompt or idle state again — your trackpad will physically buzz to let you know.

How it works
Claude Code fires Notification hooks when it needs input. HaptiClaude intercepts these events and drives the Taptic Engine through Apple's private MultitouchSupport.framework, with an NSHapticFeedbackManager fallback.

Notification Events

EventTrigger
permission_promptClaude needs approval to run a tool (Bash, Edit, etc.)
idle_promptClaude finished responding, waiting for your next message
elicitation_dialogAn MCP server is requesting user input

02 Architecture

03 The Haptic Binary

The haptic binary is a native Swift CLI tool that actuates the MacBook trackpad's Taptic Engine. It requires no permissions, entitlements, or code signing.

Usage

./haptic [pattern] [time] [--strength N] [--repeat N] [--delay MS]

Parameters

ParameterValuesDefault
patterngeneric, alignment, levelChangegeneric
timenow, default, drawCompleteddefault
--strength1 – 16 (raw actuator intensity)Pattern-dependent
--repeat1 – N (number of beats)1
--delay0 – N ms (between beats)50

Base Patterns

PatternFeelDefault Strength
genericSoft, neutral tap4
alignmentSubtle, precise click2
levelChangePronounced thud6

Dual Actuation Strategy

The binary tries two methods in order:

  1. MultitouchSupport.framework (private API) — calls MTActuatorActuate directly. Works from background processes without requiring the calling app to be frontmost.
  2. NSHapticFeedbackManager (AppKit) — public API fallback. Creates a minimal NSApplication context with .accessory activation policy to enable haptics from a CLI process.

Build from Source

swiftc haptic.swift -o haptic \
  -framework AppKit \
  -F /System/Library/PrivateFrameworks \
  -framework MultitouchSupport -O

04 Hook Configuration

HaptiClaude hooks into Claude Code's Notification event. The hook script receives JSON on stdin and triggers the haptic binary.

Settings Entry

Added to ~/.claude/settings.json by the installer:

{
  "hooks": {
    "Notification": [
      {
        "matcher": "permission_prompt|idle_prompt|elicitation_dialog",
        "hooks": [
          {
            "type": "command",
            "command": "/path/to/hooks/haptic-notify.sh",
            "timeout": 10
          }
        ]
      }
    ]
  }
}

Current Pattern Configuration

The hook fires a single pattern for all notification types:

# 5-second continuous rumble at max strength
./haptic levelChange now --strength 16 --repeat 100 --delay 50
PropertyValue
PatternlevelChange (strongest base feel)
Strength16 (maximum actuator intensity)
Repeat100 beats
Delay50ms between beats
Total duration~5 seconds

05 Pattern Composer

HaptiClaude includes a web-based visual pattern composer for designing and testing haptic patterns interactively.

Starting the Composer

cd composer
python3 server.py
# Opens at http://localhost:4780

The composer provides:

  • Three preset slots — Permission (amber), Idle (blue), MCP (purple)
  • Visual timeline — glowing pulse bars sized by strength, spaced by delay
  • Live trackpad testing — Play buttons fire real haptics via the local server
  • Pattern sampler — tap to compare Generic, Alignment, and Level Change feels
  • Export — generates the complete hook script with your custom patterns
Requirement
The local Python server (server.py) must be running for live trackpad testing. The visual preview works without it.

06 Installation

Quick Install

git clone <repo> && cd HaptiClaude_MacOs
./install.sh

The installer builds the Swift binary, tests haptic feedback, and registers the hook in ~/.claude/settings.json.

Requirements

RequirementDetails
HardwareMacBook with Force Touch trackpad (2015+) or Magic Trackpad 2+
SoftwaremacOS, Swift compiler (xcode-select --install)
Claude CodeAny version with Notification hooks support

Uninstall

./uninstall.sh

Removes the hook entry from settings. Delete the directory to fully remove.

07 Technical Notes

Why MultitouchSupport?

The public NSHapticFeedbackManager API may silently suppress haptic feedback when the calling process is not the frontmost application. Since Claude Code hooks run as background shell commands, the private MTActuatorActuate function from MultitouchSupport.framework is used as the primary actuation method — it bypasses the foreground-app check and drives the Taptic Engine directly.

Strength Scale (1–16)

The --strength parameter maps directly to the first argument of MTActuatorActuate. Apple's public API only exposes three levels (Generic ≈ 4, Alignment ≈ 2, Level Change ≈ 6), but the private API accepts values up to 16 for significantly stronger actuation.

No Permissions Required

Both NSHapticFeedbackManager and MTActuatorActuate work without special entitlements, sandbox exceptions, user permission prompts, or code signing. The binary runs as an unsigned CLI tool.
